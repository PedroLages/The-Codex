# Janitor Skill

## Purpose

Maintain The Codex without making it complicated.

Use this skill to audit, clean, and repair the system safely.

## When to Use

Use when Pedro asks:

- clean my vault
- audit my vault
- check my AI system
- find problems
- review the structure
- organize this safely
- check for messy notes

## Output Location

Reports go in:

`90. AI/generated/`

File name format:

```text
janitor-report-YYYY-MM-DD.md
```

## What To Check

Review:

* duplicate AI instructions
* outdated files
* empty generated drafts
* notes in the wrong folder
* too many tags
* missing template files
* broken links
* unclear note titles
* AI drafts accidentally placed in main notes
* source notes without possible main notes
* main notes without connections
* old references to `05. Maps of Content`
* old references to `Indexes`
* old references to `AIOS`

## Output Format

```markdown
# Janitor Report

## What Looks Good

-

## Problems Found

-

## Suggested Fixes

-

## Fixes That Need Pedro's Approval

-

## Do Not Touch

-
```

## Safety Rules

* Do not delete files.
* Do not move files without permission.
* Do not rename files without permission.
* Report first, act only after approval.
* Prefer simple fixes.
* Do not create new systems.
