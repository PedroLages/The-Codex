# Note Quality Checker Skill

## Purpose

Check whether a note is ready to become a strong main note.

This skill protects the quality of `03. Main Notes/`.

It helps Pedro improve notes without replacing his thinking.

## When To Use

Use when Pedro says:

- check this note
- is this a good main note?
- improve this atomic note
- review this note
- is this ready?
- make this note better
- quality check

## Main Note Standard

A good main note should:

- focus on one idea
- be written in Pedro's own words
- be understandable without the original source
- have a clear title
- explain why the idea matters
- include one concrete example when useful
- link to related notes
- link to source material when possible
- use tags as linked notes
- connect to a MOC when the topic is large enough

## Quality Checklist

Check:

```text
1. One idea only?
2. Clear title?
3. Written in Pedro's own words?
4. Understandable on its own?
5. Has a useful example?
6. Has connections?
7. Has source reference?
8. Has useful tag links?
9. Belongs in Main Notes?
10. Needs a MOC link?
```

## Rules

* Do not rewrite the note unless Pedro asks.
* Do not make the note longer just to make it look better.
* Preserve Pedro's voice.
* Prefer simple improvements.
* If the note is actually source material, say so.
* If the note is too broad, suggest splitting it.
