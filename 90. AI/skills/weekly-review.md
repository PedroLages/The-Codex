# Weekly Review Skill

## Purpose

Review Pedro's week as an extra lens.

This does not replace Pedro's own thinking.

It helps surface:

- what he learned
- what moved forward
- what is still open
- what should continue next week
- what may be slipping

## When to Use

Use when Pedro asks:

- weekly review
- review my week
- what did I learn this week?
- what should I focus on next week?
- summarize my learning
- check my progress

## Output Location

Default output:

`90. AI/generated/`

## Output Format

```markdown
# Weekly Review - YYYY Week WW

## What I Learned

-

## What I Built Or Practiced

-

## Best Notes Created

-

## Open Loops

-

## What Is Slipping

-

## Connections I Noticed

-

## Focus For Next Week

1.
2.
3.

## Notes To Create Or Improve

-
```

## Rules

* Be honest.
* Be practical.
* Do not overstate progress.
* Do not create fake insights.
* Use actual notes when available.
* Keep Pedro's own judgment in control.

## Memory Proposal Step

At the end of every weekly review, check whether stable memory updates are needed.

Use this workflow:

```text
Weekly Review → Memory Proposal → Pedro Approval → memory.md Update → memory-changelog.md entry
```

Look for:

* new vault decisions
* naming decisions
* AI workflow decisions
* active project changes
* long-term preferences
* learning focus changes
* outdated memories

Do not edit `90. AI/memory.md`.

Instead, create a memory update proposal in:

```text
90. AI/generated/
```

Use this filename format:

```text
memory-update-proposal-YYYY-MM-DD-HHMM.md
```

Then ask Pedro for approval.
